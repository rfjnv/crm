# Excel Import System для CRM

## 📋 Обзор

Импортер преобразует данные из Excel файлов в структурирированные записи в базе данных CRM:
- **Товары** (Products) - каталог с ценами и единицами измерения
- **Клиенты** (Clients) - список компаний и контактов
- **Сделки** (Deals) - сгруппированные по дате и клиенту
- **Платежи** (Payments) - история оплат
- **Инвентарь** (InventoryMovements) - движения товаров со склада

## 📊 Структура Excel файлов

### Поддерживаемые файлы
```
analytics_2024-12-26.xlsx  (2.5 MB, период: декабрь 2024)
analytics_2025-12-29.xlsx  (1.8 MB, период: декабрь 2025)
analytics_2026-03-17.xlsx  (373 KB, период: март 2026)
```

### Столбцы Excel (по индексам)

| Индекс | Заголовок | Назначение | Пример |
|--------|-----------|-----------|--------|
| A | дата | Дата сделки | 45644 |
| B | фирма | Имя клиента | "вм принт" |
| C | Ост-к на [дата] | Долг на начало | 784000 |
| D (№3 __EMPTY) | - | Менеджер (опционально) | "тимур" |
| E (№4) | Отпуск товара за месяц | Название товара | "лам 62 голд" |
| F (№5 __EMPTY_1) | - | Количество | 307.4 |
| G (№6 __EMPTY_2) | - | Единица измерения | "кг" |
| H (№7 __EMPTY_3) | - | Цена за единицу | 2500 |
| I (№8 __EMPTY_4) | - | Резерв | 0 |
| J (№9) | нкп | Код платежа | к/н/п/пп/ф |
| K (№10) | договор номер | Номер договора | "12345" |
| L-Z | Оплата... | Способы платежа | наличные/перечисл/QR/пластик/терминал |
| AA (№26) | Ост на [дата] | Остаток долга | 784000 |
| AB (№27) | число | День платежа | 15 |

### Коды платежей (столбец J)

```
к    = Долг (кредит)               → TRANSFER
н    = Наличные                    → CASH
н/к  = Наличные + товар в долг     → CASH
п    = Перечисление / Трансферт    → TRANSFER
п/к  = Перечисление + товары в долг → TRANSFER
пп   = Передоплата (Prepayment)    → CLICK
ф    = Фотих (специальное лицо)    → CASH
```

## 📈 Логика обработки данных

### Создание сделок

Система группирует строки в сделки по критериям:
- **[ОСНОВНОЙ]** Один клиент
- **[ОСНОВНОЙ]** Одна дата
- **[ДОП]** Разные товары = разные позиции (DealItem) в одной сделке

Пример группировки:
```
Дата: 15.03.2026, Клиент: "вм принт"
  ├─ Товар 1: "лам 62 голд" кол-во 307.4 кг @ 2500= 768500
  ├─ Товар 2: "печать" кол-во 100 шт @ 5000 = 500000
  └─ Сделка создается ОДНА с 2 позициями (DealItem)
```

### Расчет платежного статуса

```
Если остаток долга (AA) > 0  → PaymentStatus: PARTIAL / UNPAID
Если остаток долга = 0       → PaymentStatus: PAID
                             → DealStatus: CLOSED
```

### Инвентарные движения

Для каждой позиции товара создается:
- `type: OUT` (товар со склада)
- `quantity`: количество из столбца F
- `dealId`: ссылка на сделку

## 📊 Примеры обработки

### Пример 1: Товар с платежом

```
Строка:
  Клиент: "селена трейд"
  Товар: "лам 62 голд"
  Кол-во: 307.4 кг
  Цена: 2500
  Платеж: "пп" (передоплата)
  Остаток: 0

Результат:
  ✓ Product: "лам 62 голд" (unit: кг)
  ✓ Client: "селена трейд"
  ✓ Deal: "селена трейд - 2026-03-15"
    - status: CLOSED
    - amount: 768500
    - paymentStatus: PAID
  ✓ DealItem: 307.4 кг связано с товаром
  ✓ InventoryMovement: -307.4 кг (OUT)
```

### Пример 2: Долг без товара

```
Строка:
  Клиент: "жакар уз"
  Товар: [пусто]
  Платеж: "ф" (Фотих)
  Остаток: 4940000

Результат:
  ✓ Client: "жакар уз"
  ✓ Deal: "жакар уз - остаток 2026-03-15"
    - status: IN_PROGRESS
    - amount: 4940000
    - paymentStatus: UNPAID
  ✓ DealItem: [НЕ создается - нет товара]
```

## 📋 Обработка ошибок

Импортер обрабатывает ошибки gracefully:

```
❌ Пустое имя клиента → ПРОПУСК СТРОКИ
❌ Пустое имя товара (если остаток товара > 0) → ПРОПУСК позиции товара
✅ Менеджер не существует → СОЗДАНИЕ нового менеджера
✅ Клиент не существует → СОЗДАНИЕ нового клиента
✅ Товар не существует → СОЗДАНИЕ нового товара
✅ Дубликат DealItem → ПРОПУСК (игнорируется)
✅ Дубликат InventoryMovement → ПРОПУСК (игнорируется)
```

## 🔄 Расчёт долга и выручки

### Долг клиента

Для расчета ПОЛНОГО долга клиента по всему периоду:

```sql
SELECT
  SUM(closing_balance)  -- Из столбца AA
FROM deal_items
WHERE
  deal.client_id = :clientId
  AND deal.payment_method IN ('к', 'н/к', 'п/к', 'ф')
```

### Долг к оплате (не просроченный)

```sql
SELECT
  SUM(closing_balance)
FROM deal_items
WHERE
  deal.client_id = :clientId
  AND deal.payment_status != 'PAID'
```

### Выручка по дням

```sql
SELECT
  DATE(deal.created_at),
  SUM(deal_item.line_total)
FROM deal_items
GROUP BY DATE(deal.created_at)
```
