# Управление менеджерами после импорта

## 📊 Автоматическое создание менеджеров

При импорте данных из Excel автоматически создаются учетные записи менеджеров:

### Источник данных
- **Столбец D** (Excel: `__EMPTY`) - имя менеджера
- Примеры: "тимур", "дилмурод", "азамат"

### Логика создания

```
Если менеджер с таким именем существует → используется существующий
Если не существует → создается новый аккаунт
```

## 🔐 Данные для входа

### Автоматически созданные менеджеры

Для каждого менеджера из Excel создается:

| Поле | Значение | Пример |
|------|----------|--------|
| **Login** | `имя.фамилия` (без пробелов) | `тимур` → `timur` |
| **Password** | `temp123` | (хешировано) |
| **Full Name** | Из Excel | `Тимур` |
| **Role** | `MANAGER` | - |

### Default Manager

Если в строке Excel нет имени менеджера:

| Поле | Значение |
|------|----------|
| **Login** | `default-manager` |
| **Password** | `default123` |
| **Full Name** | `Default Manager` |
| **Role** | `MANAGER` |

## 🔧 Первый вход менеджера

После импорта менеджеры должны:

1. **Войти с временным паролем**
   ```
   Login: timur (или их транслитерированное имя)
   Password: temp123
   ```

2. **Сменить пароль**
   - Перейти в Профиль → Настройки
   - Изменить пароль на безопасный

## 📋 Список созданных менеджеров

После импорта посмотрите список:

### Через API
```bash
GET /api/users?role=MANAGER
Authorization: Bearer [admin_token]
```

### Через базу данных
```sql
SELECT
  id,
  login,
  full_name,
  role,
  is_active,
  created_at
FROM users
WHERE role = 'MANAGER'
ORDER BY created_at DESC;
```

## ⚙️ Управление менеджерами

### Активация/деактивация
```sql
-- Деактивировать менеджера
UPDATE users
SET is_active = false
WHERE login = 'timur';

-- Активировать обратно
UPDATE users
SET is_active = true
WHERE login = 'timur';
```

### Сброс пароля
Админ может сбросить пароль менеджера через UI или API:
```bash
POST /api/users/:userId/reset-password
Authorization: Bearer [admin_token]
```

### Изменение роли
```sql
UPDATE users
SET role = 'ADMIN'
WHERE login = 'timur';
```

## 🔍 Проверка после импорта

### 1. Проверьте созданных менеджеров
```bash
cd backend
npm run db:studio
# Откроется Prisma Studio → Users → Фильтр: role = MANAGER
```

### 2. Список уникальных менеджеров в Excel
Перед импортом можно проверить:
```javascript
const XLSX = require('xlsx');
const wb = XLSX.readFile('analytics_2026-03-17.xlsx');
const ws = wb.Sheets[wb.SheetNames[0]];
const rows = XLSX.utils.sheet_to_json(ws);
const managers = [...new Set(rows.map(r => r['__EMPTY']).filter(Boolean))];
console.log('Менеджеры:', managers);
```

### 3. Тестовый вход
Попробуйте войти как каждый менеджер:
```
URL: https://your-app.com/login
Login: timur
Password: temp123
```

## 🚨 Важно!

### Безопасность
1. ✅ Пароли **хешируются** через bcrypt (saltRounds: 12)
2. ✅ Нельзя восстановить оригинальный пароль из БД
3. ⚠️ Менеджеры **ОБЯЗАНЫ** сменить временный пароль!

### После импорта обязательно:
- [ ] Уведомить всех менеджеров о создании аккаунтов
- [ ] Предоставить временные пароли
- [ ] Потребовать смену пароля при первом входе
- [ ] Проверить что все менеджеры активны (`is_active = true`)

## 🔄 Повторный импорт

При повторном импорте:
- ✅ Существующие менеджеры **НЕ** дублируются
- ✅ Используются существующие учетные записи
- ✅ Пароли существующих менеджеров **НЕ** изменяются

## 💡 Советы

### Транслитерация имен
Система автоматически преобразует кириллицу:
```
тимур → timur
дилмурод → dilmurod
азамат → azamat
фарход → farxod
```

Если есть пробелы:
```
"Тимур Алиев" → "timur.aliev"
```

### Конфликты логинов
Если два менеджера с одинаковым именем:
```
Первый: timur
Второй: timur (уже занят) → timur-1234567890 (с timestamp)
```
