# 🚀 Быстрый старт импорта Excel

## 1️⃣ Убедитесь, что Excel файлы находятся в корне проекта

```bash
# Проверка наличия файлов
ls -lh analytics_*.xlsx
```

Файлы должны быть в: `c:\Users\Noutbuk savdosi\CRM\`

```
✓ analytics_2024-12-26.xlsx
✓ analytics_2025-12-29.xlsx
✓ analytics_2026-03-17.xlsx
```

## 2️⃣ Перейдите в директорию backend

```bash
cd backend
npm install  # (если требуется)
```

## 3️⃣ Запустите импорт

```bash
npm run import-excel
```

## 🔍 Что было импортировано

После завершения в БД будут созданы:

### 📦 **Товары** (Products)
- Все уникальные товары из Excel
- С единицами измерения и ценами
- Доступны в системе для заказов

### 👥 **Клиенты** (Clients)
- Все компании из столбца "фирма"
- Связаны с менеджерами
- Имеют историю сделок

### 📋 **Сделки** (Deals)
- Сгруппированы по дате и клиенту
- Содержат позиции товаров (DealItems)
- Отслеживают платеж и остаток долга

### 💰 **Платежи** (Payments)
- История платежей от клиентов
- Связана с каждой сделкой

### 📦 **Инвентарь** (InventoryMovements)
- Отслеживание товаров со склада
- Связано с каждой сделкой покупки

## ⚠️ Возможные проблемы

### "Error: ENOENT: no such file"
**Проблема**: Файлы Excel не найдены
**Решение**: Убедитесь, что файлы в корне проекта (не в backend/)

### "Database connection failed"
**Проблема**: Нет подключения к БД
**Решение**: Проверьте переменную `DATABASE_URL` в `.env`

### "Transaction failed"
**Проблема**: Конфликт уникальных ключей
**Решение**: Импортер разработан для идемпотентности - повторный запуск пропустит существующие записи

## ✅ Проверка данных


## 📝 Логирование

Импортер создает детальные логи:

```
✅ Импортировано = строки успешно обработаны
⏭️  Пропущено = строки с ошибками или пустые
⚠️  Строка N: [причина] = конкретная ошибка
```

Первые 5 ошибок выводятся в консоль, остальные игнорируются.

## 🔄 Повторный импорт

Можно запустить импорт несколько раз:
- ✓ Новые записи будут добавлены
- ✓ Существующие записи не будут продублированы
- ✓ Обновления возможны вручную через UI

## 💡 Советы

1. **Первый раз**: Может занять 2-5 минут в зависимости от размера БД
2. **Мониторинг**: Смотрите логи консоли для отслеживания прогресса
3. **Данные**: После импорта - перезагрузите приложение чтобы увидеть новые данные
4. **Поддержка**: Если вопросы - смотрите [EXCEL_IMPORT_GUIDE.md](./EXCEL_IMPORT_GUIDE.md)
